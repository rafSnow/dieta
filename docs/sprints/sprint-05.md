# Sprint 5 - Dashboard

## Objetivo
Dar visibilidade ao que já foi construído nas Sprints anteriores, fornecendo um resumo diário e gráficos de evolução.

## Escopo (Product Backlog)
- **E8-01:** Dashboard diário com consumido vs. meta (Cards/barras de progresso).
- **E7-04:** Registrar peso ao longo do tempo (Form simples + listagem).
- **E8-03:** Gráfico de evolução de peso usando Recharts (Gráfico de linha).

## Entregáveis
- Tela inicial nova (`/`) focada em ser um Dashboard.
- Rota dedicada para o Registro Diário movida para `/diario`.
- Integração do componente de Peso com o repositório `historicoPeso` usando Dexie.
- Instalação e uso do `recharts` para o histórico de peso.
